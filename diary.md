## 2026-06-18

这几天 Xbox 手柄和蓝牙适配器出了 bug。手柄有线连不上，只能蓝牙；蓝牙适配器驱动加载不了，显示驱动程序错误。家里的台式机和幻 14 笔记本都有同样的问题。两台机器都是用拼多多买的系统 U 盘装的系统，所以我觉得 100% 是这个系统 U 盘的问题。

台式机我用 U 盘自带的系统重装功能重装了一遍，所有问题都解决了。但是笔记本没法重装，因为 U 盘的重装功能里下载不了镜像，不知道什么原因。然后我用原来的镜像直接重装，没解决，还是同样的问题，说明这个镜像本身就有 bug。

后来用 Media Creation Tool 更新了系统，Xbox 驱动好了，有线连接正常了，蓝牙适配器也能加载驱动了。但是又冒出新的问题：微软商店打不开了，一点就闪退，里面下载的软件也全都这样。试了重装商店、Media Creation Tool 更新、删注册表、sfc /scannow、wsreset，全都没用。

让 Claude 帮忙查了一下，定位到问题。事件日志显示错误码 2147942402，对应"文件不存在"。商店的 exe 文件在硬盘上，说明缺了依赖。查了运行时配置，商店需要 .NET 10.0.9，系统里只有 .NET 8.0。装了 .NET 10 之后，商店从闪退变成了打不开，因为包没注册到当前用户。试了重新注册，没用。wsreset -i 直接卡死。最后跑了 DISM /RestoreHealth，DISM 刚跑上，Windows 弹通知说微软商店安装成功，能打开了。这时候总结是两个 bug 叠加：缺 .NET 10 运行时 + 商店包注册损坏，DISM 修好了底层组件存储。

商店修好了，但里面下载的应用全部打不开，弹窗都报找不到 WindowsApps 目录中的 exe。排查了好几个小时：

- 文件在硬盘上，权限正常，包也注册给当前用户了，状态 OK
- 事件日志显示所有应用激活都死在 COM ActivateExtension 阶段，"系统找不到指定的文件"
- 查注册表 `HKLM\SOFTWARE\Classes\ActivatableClasses\Package`，里面只有 2 个系统包，没有任何第三方应用的 COM 注册——正常系统这里应该有几十个
- 用 Add-AppxPackage 批量重新注册所有包，COM 注册表完全没变化，写不进去
- 又跑了 DISM /StartComponentCleanup 和 /RestoreHealth，然后重启

总结：拼多多 U 盘的系统镜像烂得太深，缺 .NET 运行时只是表面，底层 COM 注册写入层也是坏的。DISM 修好了商店的注册，但这次不确定能不能修好 COM 层。如果还不行，最后只能去微软官网下 ISO 做原地升级（保留文件和应用），把系统组件全换成官方的。
