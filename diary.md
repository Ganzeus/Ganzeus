## 2026-06-18

这几天 Xbox 手柄和蓝牙适配器出了 bug。手柄有线连不上，只能蓝牙；蓝牙适配器驱动加载不了，显示驱动程序错误。家里的台式机和幻 14 笔记本都有同样的问题。两台机器都是用拼多多买的系统 U 盘装的系统，所以我觉得 100% 是这个系统 U 盘的问题。

台式机我用 U 盘自带的系统重装功能重装了一遍，所有问题都解决了。但是笔记本没法重装，因为 U 盘的重装功能里下载不了镜像，不知道什么原因。然后我用原来的镜像直接重装，没解决，还是同样的问题，说明这个镜像本身就有 bug。

后来用 Media Creation Tool 更新了系统，Xbox 驱动好了，有线连接正常了，蓝牙适配器也能加载驱动了。但是又冒出新的问题：微软商店打不开了，一点就闪退，里面下载的软件也全都这样。

试过的方法：

- 重装微软商店 -> 没用
- 用 Media Creation Tool 更新系统 -> 没用
- 彻底删除微软商店，包括注册表 -> 没用
- `sfc /scannow` 修复系统 -> 这个命令今天运行成功了（之前 U 盘装出来的系统连这个命令都跑不了），但对闪退没用
- `wsreset` -> 没用

反正试了一圈，全都没解决。

## 2026-06-18 续

后来让 Claude 查了系统，定位到了问题。

事件日志显示每次打开商店都报错 2147942402，对应"文件不存在"。商店的 exe 文件在硬盘上，说明启动时缺少了某个依赖。

商店的运行时配置显示它需要 .NET 10.0.9，但系统里只有 .NET 8.0。装了 .NET 10 之后，商店从闪退变成了打不开，报"找不到文件"。这时候问题是商店包没有注册到当前用户。

试了重新注册商店包，没用。试了 wsreset -i，命令卡死。最后跑了 DISM /RestoreHealth，这个命令刚执行，Windows 弹通知说微软商店安装成功。打开商店，能进了。

复盘：U 盘系统镜像有问题，两个 bug 叠加——缺少 .NET 10 运行时 + 商店包注册损坏。前者导致闪退，后者导致修复手段无效。DISM 修了底层组件存储，顺带恢复了商店注册。
