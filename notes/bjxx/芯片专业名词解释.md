### 🔷 芯片形态相关

**SoC（System on Chip，片上系统）**
 将CPU、内存控制器、接口、AI加速器等所有功能模块集成在一块芯片上。类比：手机里的高通骁龙芯片就是典型SoC。

**Chiplet（小芯片/裸片拼接）**
 将一个大芯片拆分成多个小的"裸片（Die）"，通过高速互联拼在同一封装内。好处是可以混合不同工艺节点，降低良率损失；缺点是互联设计复杂。文档中探索了"NPU裸片+CPU裸片"和"双NPU裸片"两种Chiplet方案。

**Die（裸片）**
 从晶圆上切割下来的单颗芯片本体，未经封装的原始硅片。

**良率（Yield）**
 晶圆上能正常工作的Die的比例。Die面积越大，良率越低，因此Chiplet方案可以把大Die切小，提升良率。

**流片（Tape-out）**
 将芯片设计文件交给晶圆厂正式生产，是芯片研发的关键里程碑，费用极高（12nm Full Mask流片费用约数千万美元）。

**MPW（Multi-Project Wafer，多项目晶圆）**
 多个设计共用一块晶圆分摊成本，常用于验证，但产量极少。Full Mask则是独占整块掩模版，用于正式量产。

**TSMC 12FFC**
 台积电（Taiwan Semiconductor Manufacturing Company）的12nm FinFET Compact工艺节点，是本芯片的制造工艺。

### 🔷 计算性能相关

**TOPS（Tera Operations Per Second，万亿次操作每秒）**
 AI芯片算力的衡量单位。文档中启明940目标是INT8下32 TOPS，INT4下64 TOPS。

**INT4 / INT8 / INT16 / FP16 / BF16**
 不同的数值精度格式：

- **INT4/8/16**：4/8/16位整数，用于量化推理，精度低但计算快、功耗低
- **FP16**：16位浮点数（IEEE半精度），精度较高
- **BF16（Brain Float 16）**：Google为AI训练设计的16位浮点格式，指数位更多（更大的数值范围），尾数位较少，由Google Brain提出

**MAC（Multiply-Accumulate，乘累加）**
 AI计算最核心的基本操作：`result += A × B`。卷积神经网络本质上就是海量MAC运算。

**TOPS/W（能效比）**
 每瓦特功耗能提供的计算能力，是衡量AI芯片"省不省电"的关键指标。文档目标是2 TOPS/W。

### 🔷 NPU架构相关

**NPU（Neural Processing Unit，神经网络处理单元）**
 专门为神经网络推理/训练设计的处理器，相比CPU/GPU在AI任务上有更高的能效。

**MUSE（Multi-grained Sparsity Engine）**
 启明930及之前版本使用的NPU核心名称，主打稀疏计算（利用神经网络中大量零值跳过计算）。

**ZEUS（Zippy Engine for Universal AI Speedup）**
 启明940的新NPU核心名称，意为"快速的通用AI加速引擎"，放弃稀疏，转向通用性和可扩展性。

**Tensor Core（张量核心）**
 ZEUS架构中的基本计算节点，相当于之前的MUSE核。每个Tensor Core内部包含4×4个PE，以2D-Mesh互联。全芯片有4×4=16个Tensor Core。

**PE（Processing Element，处理单元）**
 最底层的计算单元，包含ALU（算术逻辑单元）、本地缓存（AL1/WL1）、BN模块等。每个PE有8个lane，每个lane包含8个MAC。

**Lane**
 PE内部并行计算的一路通道，可以理解为"并行流水线"。

**Vector Processor（向量处理器）**
 每个Tensor Core中新增的模块，负责处理池化、上采样、Element-wise运算等非卷积操作，是相比930版本的重要新增。

**ALU（Arithmetic Logic Unit，算术逻辑单元）**
 执行加减乘除等基本运算的硬件单元。

**BN模块（Batch Normalization模块）**
 硬件中专门执行批归一化的模块，通常与卷积融合执行以节省时间。

### 🔷 存储层次相关

**SRAM（Static Random Access Memory，静态随机存储器）**
 速度最快的片上存储器，用于缓存（Cache），但面积大、功耗高。

**L1 / L2 / L3 Buffer（一/二/三级缓存）**
 芯片内部的多级缓存结构，越靠近计算单元速度越快但容量越小：

- **L1（WL1/AL1/OL1）**：在PE内部，存放当前计算所需的权重（Weight）和激活值（Activation）
- **L2 Buffer**：每个Tensor Core的本地缓存，1MB
- **L3（Unified Shared Buffer）**：所有Tensor Core共享的全局缓存，类似NUMA架构

**LPDDR4X**
 低功耗DDR4扩展内存，常用于移动/边缘设备。文档中规格为256bit位宽×8组，带宽约1066 Gbps。

**GDDR6**
 图形DDR6内存，带宽更高，用于AI加速子系统（NPU直接访问）。

**DRAM（Dynamic Random Access Memory）**
 动态随机存取存储器，即通常说的内存，需要不断刷新，价格低、容量大但速度比SRAM慢。

**DDR（Double Data Rate）**
 双倍数据速率内存，在时钟上升沿和下降沿都传输数据，倍增带宽。

**Weight Buffer**
 专门存储神经网络权重参数的缓冲区。

**Ping-Pong Buffer（乒乓缓冲）**
 双缓冲技术：一块缓冲区在写入新数据的同时，另一块供计算读取，交替使用以实现流水线，消除等待时间。

**NUMA（Non-Uniform Memory Access，非一致内存访问）**
 多核系统中，不同核心访问不同内存区域的速度不同的架构。文档中L3 Buffer以类NUMA方式分布在4×4 Mesh中。

### 🔷 互联与总线相关

**NoC（Network on Chip，片上网络）**
 芯片内部用于连接各模块的通信网络，类比"芯片内部的互联网"。

**2D-Mesh（二维网格拓扑）**
 NoC的一种拓扑结构，所有节点排成二维网格，每个节点与上下左右四个邻居相连。文档中NPU核心按4×4的2D-Mesh排列。

**AXI（Advanced eXtensible Interface）**
 ARM定义的高性能片上总线协议（AMBA协议族之一），广泛用于SoC设计，支持读写并发、乱序传输。

**FlexNoc（Arteris FlexNoc）**
 Arteris公司的商业片上网络IP，文档中用于SoC系统总线互联（外购）。

**Innolink**
 文档中使用的Die-to-Die高速互联接口，用于连接两个芯片裸片（实现Chiplet方案）。

**D2D（Die-to-Die）**
 裸片间直接互联接口，Chiplet方案的核心技术。

**PCIe（Peripheral Component Interconnect Express）**
 高速串行计算机扩展总线标准，常用于连接GPU、SSD等高速设备。文档中支持PCIe 4.0 x16，带宽256Gbps。RC指根复合体（主控端），EP指端点（从设备端）。

**RGMII（Reduced Gigabit Media Independent Interface）**
 千兆以太网的物理层接口标准。

**MIPI（Mobile Industry Processor Interface）**
 移动设备行业标准接口联盟定义的接口协议，常用于摄像头（MIPI CSI）和显示（MIPI DSI）连接。

**USB（Universal Serial Bus）**
 通用串行总线，文档支持USB 3.1（10Gbps）。

**I2C / SPI / UART / I2S / GPIO**
 常见低速接口：

- **I2C**：两线制串行总线，用于连接传感器等低速设备
- **SPI**：四线制高速串行接口
- **UART**：通用异步收发传输器，用于串口通信
- **I2S**：用于音频数据传输的串行总线
- **GPIO**：通用输入输出引脚

**EMMC（Embedded Multi Media Card）**
 嵌入式存储芯片，类似手机内置存储。

**QSPI Flash**
 四路SPI Flash存储器，用于存储启动引导程序（Bootloader）。

**SD 6.0**
 SD存储卡协议的第六代标准，支持更高传输速率。

### 🔷 处理器与子系统相关

**Host CPU（主机CPU）**
 SoC中负责运行操作系统、调度任务的主处理器。文档中使用16核ARM A76，频率1.8~2.1GHz。

**ARM A76（ARM Cortex-A76）**
 ARM的高性能CPU核心设计（用于授权给芯片厂商使用），常见于2019-2020年旗舰手机。

**MCU（Micro-Controller Unit，微控制器）**
 低功耗小型控制器，文档中使用ARM China（安谋中国）的"星辰"系列，负责系统控制和低功耗管理（如休眠唤醒）。

**ARM China 星辰系列**
 安谋科技（ARM中国合资公司）自主研发的MCU IP产品线。

**ISP（Image Signal Processor，图像信号处理器）**
 负责将摄像头原始RAW数据转换为可用图像的专用处理器（去噪、白平衡、HDR等）。文档中启明940本身不集成ISP，依赖外部芯片。

**CODEC（编解码器）**
 视频/图像的压缩编码和解压解码模块。文档中使用ARM LINLON-V5 IP，支持4K@240fps H.264/265解码。

**H.264 / H.265（HEVC）**
 视频压缩编码标准，H.265压缩率约是H.264的两倍。

**ARM LINLON-V5**
 ARM授权的视频编解码IP。

**DMA（Direct Memory Access，直接内存访问）**
 让外设直接读写内存，不需要CPU参与，释放CPU算力。文档中LOAD DMA和STORE DMA分别负责NPU的数据搬入和搬出。

**WDT（Watchdog Timer，看门狗定时器）**
 硬件计时器，若系统软件崩溃未能定时"喂狗"，则自动复位系统。

### 🔷 量化与神经网络相关

**量化（Quantization）**
 将神经网络中的浮点数（FP32）压缩为低精度整数（INT8/INT4），大幅降低存储和计算开销，代价是少量精度损失。

**PTQ（Post-Training Quantization，训练后量化）**
 训练完成后直接对模型进行量化，无需重新训练，操作简便但精度损失较训练感知量化稍大。

**非对称量化（Asymmetric Quantization）**
 用 `scale`（缩放因子）和 `zero_point`（零点偏移）两个参数做量化，能更好适应数据分布不均匀的情况（如ReLU后的激活值全为正数）。

**对称量化（Symmetric Quantization）**
 只用 `scale` 参数量化，零点固定为0，实现简单但对非对称分布精度稍差。

**通道级量化（Channel-wise Quantization）**
 为每个输出通道单独设置一组量化参数，精度高。文档中权重（Weight）采用此方式。

**张量级量化（Tensor-wise Quantization）**
 整个张量共用一组量化参数，参数少、实现简单。文档中激活值（Activation）采用此方式。

**scale（缩放因子）/ zero_point（零点）**
 非对称量化的两个核心参数，用于在浮点域和整数域之间做映射。

**Activation（激活值）**
 神经网络中层与层之间传递的特征图数据，是随输入动态变化的。

**Weight（权重）**
 神经网络中训练固定下来的参数，推理时不变。

**BN（Batch Normalization，批归一化）**
 训练中常用的归一化技术，将每层输出归一化到标准分布，加速训练、提升泛化。公式形式为 `Y = K·X + B`，推理时可与卷积层融合。

**反量化（Dequantization）**
 将整数计算结果还原回近似的浮点值，通常在BN之前执行。

**层融合（Layer Fusion）**
 将多个连续的算子（如Conv + BN + ReLU）合并为一个硬件操作，减少中间数据读写，大幅提升效率。

### 🔷 神经网络算子相关

**Conv2d（二维卷积）**
 最核心的神经网络算子，用滑动卷积核提取特征。

**Depthwise Conv（深度可分离卷积）**
 MobileNet的核心技术，将标准卷积拆分为深度卷积+逐点卷积，大幅减少计算量。

**Group Conv（组卷积）**
 将输入输出通道分成若干组，每组独立做卷积，减少参数量。AlexNet最早使用。

**Dilated Conv / Atrous Conv（空洞卷积）**
 在卷积核元素间插入空洞，扩大感受野而不增加参数，常用于语义分割。

**ConvTranspose（转置卷积/反卷积）**
 上采样操作，常用于语义分割和生成模型的解码器部分。

**Conv3d（三维卷积）**
 处理视频等三维数据的卷积，增加了时间维度。

**Linear / InnerProduct / FC（全连接层）**
 每个输入节点与每个输出节点都相连，常用于分类头部。

**Embedding（词嵌入层）**
 将离散词汇映射为连续向量的层，NLP模型的基础组件。

**ReLU / LeakyReLU / PReLU / ReLU-X**
 激活函数系列：

- **ReLU**：`max(0, x)`，最简单的激活函数
- **LeakyReLU**：负数部分有微小斜率，避免"神经元死亡"
- **PReLU**：斜率可学习的LeakyReLU
- **ReLU-X（ReLU6）**：上限裁剪为X（通常为6），适合量化

**Sigmoid / Tanh / Erf / GELU / GEGLU**
 其他激活函数：

- **Sigmoid**：输出(0,1)区间，用于二分类或门控
- **Tanh**：输出(-1,1)，用于RNN
- **Erf**：误差函数
- **GELU**：Gaussian Error Linear Unit，Transformer中常用
- **GEGLU**：GELU的门控变体，用于部分大语言模型

**Softmax**
 将向量归一化为概率分布，常用于分类输出层。

**Maxpool / AvgPool / AdaptiveAvgPool（池化层）**
 通过取最大值/平均值降低特征图尺寸，减少计算量。

**BatchNorm / LayerNorm（归一化层）**

- **BatchNorm**：对一批样本的同一通道做归一化
- **LayerNorm**：对单个样本的所有通道做归一化，Transformer常用

**Upsample（上采样）**
 增大特征图尺寸，常用模式有最近邻（Nearest）和双线性插值（Bilinear）。

**Element-wise Add/Mul（逐元素加/乘）**
 对两个形状相同的张量逐元素操作，如ResNet的残差连接。

**Permute / Transpose / Reshape / Flatten / Concat / Split / Squeeze / Unsqueeze**
 维度变换类算子，不涉及计算，主要改变数据的存储排列方式，通常由编译器处理。

**Gather / Scatter**
 按索引从张量中取元素/写元素，常用于Embedding查表。

**Einsum（爱因斯坦求和约定）**
 用简洁符号描述复杂张量运算（如矩阵乘、批量矩阵乘等），Transformer中常见。

**Dropout**
 训练时随机丢弃神经元，防止过拟合。推理时无效，文档明确标注"Inference没有该算子"。

### 🔷 指令架构相关

**指令集（ISA，Instruction Set Architecture）**
 定义处理器能执行的操作的规范。文档要求"后向兼容"，即新一代芯片的指令集能运行为老芯片写的程序。

**乱序发射（Out-of-Order Issue）**
 不按程序顺序发射指令，充分利用硬件并行性。文档中通过"信箱（Mailbox）计数器"机制管理指令间依赖。

**收信位 / 寄信位**
 ZEUS-v2自定义的指令依赖表示方式：收信位表示"我需要等谁完成"，寄信位表示"我完成后要通知谁"。

**指令类型（LD/ST/LW/SI/SO/DT/PE/VP/SYNC）**

- **LD**：从DDR加载数据到L2
- **LW**：从DDR或L2加载权重到WL1
- **ST**：从L2存储数据到DDR
- **SI（Sync In）/ SO（Sync Out）**：核间同步与数据传输
- **DT（Data Transform）**：维度变换
- **PE**：卷积/FC计算
- **VP（Vector Processor）**：池化等向量运算

### 🔷 数据流与映射相关

**WS（Weight Stationary，权重驻留）**
 数据流策略：将权重固定在计算单元的本地存储中不动，不断流入不同的激活值进行复用。减少权重的重复读取。

**Partition（分割/映射）**
 将神经网络层的计算任务分配到多个PE/Core上并行执行的方式。文档中定义了9种Partition模式（WH、WC、NH、NC、KH、KC等）。

**Layer Pipeline（层流水线）**
 多个网络层同时在不同核上并行流水执行，提升吞吐量。适用于小网络且权重能全部放在片上存储的情况。

**NHWC / NCHW**
 神经网络中Tensor的维度排列方式：N=Batch，H=Height，W=Width，C=Channel。不同框架默认不同，对硬件访存效率影响很大。

**Tile（数据块）**
 将大规模数据分块处理的单位，是AI芯片编译器做内存优化的核心概念。

**Padding（填充）**
 在输入特征图边缘填充零值，保持卷积输出尺寸或满足算法需求。

**Stride（步长）**
 卷积核滑动的步幅，影响输出特征图大小。
