# 芯片专业名词解释

> ⚠ **关于启明系列的重要澄清**（阅读前务必明确）：
>
> 公司的 NPU 产品线是 **启明（Qiming）系列**：910 → 920 → 930 → **935** → 940。
>
> | 芯片 | NPU 核架构 | 软件栈（编译器后端） |
> |---|---|---|
> | 910 / 920 / 930 | **MUSE v1/v2**（主打稀疏） | 老链路（930_sdk 等） |
> | **935** | **Zeus V2** | **`assembler` 项目（你当前的目标）** |
> | 940 | **Zeus V3** | `zeusv3_backend`（新 MLIR+Triton 链路） |
>
> **935 和 940 是两代不同的芯片**，架构细节差异较大。本词典中：
> - 涉及具体数字（如 32 TOPS、4×4 Tensor Core、16 个 Tensor Core 等）的条目，**多数来自 940 的 spec 文档**，不是 935 的规格。
> - **935 的真实硬件参数**请以 [assembler_learning_plan.md](assembler_learning_plan.md) 和项目代码 (`assembler/cfg/default.py`) 为准。
> - 本词典涉及 935/940 差异的地方都做了标注。

### 🔷 芯片形态相关

**SoC（System on Chip，片上系统）**
 将CPU、内存控制器、接口、AI加速器等所有功能模块集成在一块芯片上。类比：手机里的高通骁龙芯片就是典型SoC。

**Chiplet（小芯片/裸片拼接）**
 将一个大芯片拆分成多个小的"裸片（Die）"，通过高速互联拼在同一封装内。好处是可以混合不同工艺节点，降低良率损失；缺点是互联设计复杂。文档中探索了"NPU裸片+CPU裸片"和"双NPU裸片"两种Chiplet方案。

**Die（裸片）**
 从晶圆上切割下来的单颗芯片本体，未经封装的原始硅片。

**良率（Yield）**
 晶圆上能正常工作的Die的比例。Die面积越大，良率越低，因此Chiplet方案可以把大Die切小，提升良率。

**流片（Tape-out）**
 将芯片设计文件交给晶圆厂正式生产，是芯片研发的关键里程碑，费用极高（12nm Full Mask流片费用约数千万美元）。

**MPW（Multi-Project Wafer，多项目晶圆）**
 多个设计共用一块晶圆分摊成本，常用于验证，但产量极少。Full Mask则是独占整块掩模版，用于正式量产。

**TSMC 12FFC**
 台积电（Taiwan Semiconductor Manufacturing Company）的12nm FinFET Compact工艺节点，是本芯片的制造工艺。

### 🔷 计算性能相关

**TOPS（Tera Operations Per Second，万亿次操作每秒）**
 AI 芯片算力的衡量单位。注意：**启明 935**（当前 assembler 项目目标，Zeus V2 架构）和**启明 940**（新一代，Zeus V3 架构）算力规格不同。本词典中出现的"32 TOPS / 64 TOPS"等具体数字来自 **940 spec 文档**，不是 935 的规格——935 的算力请参考项目代码配置，不要混淆。

**INT4 / INT8 / INT16 / FP16 / BF16**
 不同的数值精度格式：

- **INT4/8/16**：4/8/16位整数，用于量化推理，精度低但计算快、功耗低
- **FP16**：16位浮点数（IEEE半精度），精度较高
- **BF16（Brain Float 16）**：Google为AI训练设计的16位浮点格式，指数位更多（更大的数值范围），尾数位较少，由Google Brain提出

**MAC（Multiply-Accumulate，乘累加）**
 AI计算最核心的基本操作：`result += A × B`。卷积神经网络本质上就是海量MAC运算。

**TOPS/W（能效比）**
 每瓦特功耗能提供的计算能力，是衡量AI芯片"省不省电"的关键指标。文档目标是2 TOPS/W。

**FLOPS（每秒浮点运算次数，Floating Point Operations Per Second）**
 衡量浮点计算能力的单位。GFLOPS = 十亿次/秒，TFLOPS = 万亿次/秒。TOPS 衡量整数算力，FLOPS 衡量浮点算力，两者不能直接比较。

**GEMM（通用矩阵乘法，General Matrix Multiplication）**
 形如 C = A×B + C 的矩阵乘法运算，是深度学习最核心的计算原语，占大模型计算量的 80% 以上。Prefill 阶段主要做 GEMM。

**GEMV（矩阵向量乘法，General Matrix-Vector Multiplication）**
 矩阵与向量的乘法，是 Decode 阶段（batch=1）的主要计算模式。计算量小但要读取全部权重，严重受内存带宽限制。

**Throughput（吞吐量）**
 单位时间内系统完成的总工作量。AI 推理中常用 tokens/s（每秒生成的 token 数）或 images/s 衡量。

**Latency（延迟）**
 完成单次请求所需的时间。与吞吐量是一对矛盾指标——加大 batch 提升吞吐量，但单次请求延迟会增加。

**Bandwidth（带宽）**
 数据传输通道的最大传输速率，单位通常是 GB/s。内存带宽（DRAM → L2 → WL1 这条路的速度上限）是 AI 加速器性能的关键瓶颈之一。

**Roofline Model（屋顶线模型）**
 分析算子性能瓶颈的可视化工具。以算术强度（FLOP/Byte，每搬 1 字节数据做多少次运算）为横轴，纵轴为实际吞吐。屋顶线以下的点是实际算子——靠左的受限于带宽，靠右的受限于算力。

**Compute-bound（计算瓶颈）**
 算子的性能受限于计算单元的算力，而非内存带宽。Prefill 阶段（大矩阵 × 大矩阵）通常是 Compute-bound。

**Memory-bound（访存瓶颈）**
 算子的性能受限于内存带宽（数据搬不过来），计算单元大量空闲等待数据。Decode 阶段（每次只算一行 × 矩阵）通常是 Memory-bound。

**Arithmetic Intensity（算术强度/计算密度）**
 每搬运 1 字节数据所执行的运算次数（FLOP/Byte）。是 Roofline Model 的横坐标——低算术强度 = Memory-bound，高算术强度 = Compute-bound。

### 🔷 NPU架构相关

**NPU（Neural Processing Unit，神经网络处理单元）**
 专门为神经网络推理/训练设计的处理器，相比CPU/GPU在AI任务上有更高的能效。

**MUSE（Multi-grained Sparsity Engine）**
 启明 **910 / 920 / 930** 系列使用的 NPU 核架构名称，主打稀疏计算（利用神经网络中大量零值跳过计算）。是 Zeus 之前的上一代架构。

**ZEUS（Zippy Engine for Universal AI Speedup）**
 从启明 **935 开始使用**的新一代 NPU 核架构名称，意为"快速的通用 AI 加速引擎"，放弃稀疏，转向通用性和可扩展性。**包含两代**：**Zeus V2**（对应 935，当前 assembler 项目目标） 和 **Zeus V3**（对应 940，走新的 `zeusv3_backend` / MLIR+Triton 链路）。两代架构细节有较大差异，不能混为一谈。

> ⚠ 以下 Tensor Core / PE / Lane / Vector Processor 等条目的**具体数字（如 4×4、16 个等）来自 940（Zeus V3）spec 文档**，不是 935 的规格。**935 的实际硬件参数请以 [assembler_learning_plan.md](assembler_learning_plan.md) Part 2 和项目代码为准**。

**Tensor Core（张量核心）**【主要用于描述 940 的层次结构】
 Zeus 架构中的基本计算节点分组概念。在 **940（Zeus V3）** 中，每个 Tensor Core 内部含 4×4 PE 阵列，以 2D-Mesh 互联，全芯片 4×4 = 16 个 Tensor Core。**935（Zeus V2）没有这种"Tensor Core"的两级层次**，它的组织方式更简单（直接是若干 core，每个 core 内含一个 PE Array）。

**PE（Processing Element，处理单元）**
 最底层的计算单元，含乘加器阵列 + 本地存储（AL1/WL1/OL1）+ BN 模块等。**935 和 940 的 PE 内部结构不同**：
 - **935（Zeus V2）**：每个 core 含一个 8×8 PE 阵列，每个 PE 是 8-wide 向量 MAC 单元（INT8 乘法 + FP32 累加）。详见 [assembler_learning_plan.md](assembler_learning_plan.md) Part 1.2。
 - **940（Zeus V3）**：按 Tensor Core 组织，每个 PE 含 8 个 lane，每个 lane 8 个 MAC。

**Lane**
 PE 内部并行计算的一路通道，可理解为"一路并行流水线"。**940 中每个 PE 有 8 个 lane**（935 的内部组织不同，不用 lane 这个层次名）。

**Vector Processor（向量处理器）**
 处理池化、上采样、Element-wise 运算、归一化、激活函数等**非矩阵类计算**的独立单元。在 935 和 940 中都存在，是相比 930（MUSE 架构）的重要新增——之前 MUSE 里这类运算处理能力有限。详见 [assembler_learning_plan.md](assembler_learning_plan.md) 2.2 节 VP 部分。

**ALU（Arithmetic Logic Unit，算术逻辑单元）**
 执行加减乘除等基本运算的硬件单元。

**BN模块（Batch Normalization模块）**
 硬件中专门执行批归一化的模块，通常与卷积融合执行以节省时间。

### 🔷 存储层次相关

**SRAM（Static Random Access Memory，静态随机存储器）**
 速度最快的片上存储器，用于缓存（Cache），但面积大、功耗高。

**L1 / L2 / L3 Buffer（一/二/三级缓存）**
 芯片内部的多级缓存结构，越靠近计算单元速度越快但容量越小：

- **L1（WL1/AL1/OL1）**：在PE内部，存放当前计算所需的权重（Weight）和激活值（Activation）
- **L2 Buffer**：每个Tensor Core的本地缓存，1MB
- **L3（Unified Shared Buffer）**：所有Tensor Core共享的全局缓存，类似NUMA架构

**LPDDR4X**
 低功耗DDR4扩展内存，常用于移动/边缘设备。文档中规格为256bit位宽×8组，带宽约1066 Gbps。

**GDDR6**
 图形DDR6内存，带宽更高，用于AI加速子系统（NPU直接访问）。

**DRAM（Dynamic Random Access Memory）**
 动态随机存取存储器，即通常说的内存，需要不断刷新，价格低、容量大但速度比SRAM慢。

**DDR（Double Data Rate）**
 双倍数据速率内存，在时钟上升沿和下降沿都传输数据，倍增带宽。

**Weight Buffer**
 专门存储神经网络权重参数的缓冲区。

**Ping-Pong Buffer（乒乓缓冲）**
 双缓冲技术：一块缓冲区在写入新数据的同时，另一块供计算读取，交替使用以实现流水线，消除等待时间。

**NUMA（Non-Uniform Memory Access，非一致内存访问）**
 多核系统中，不同核心访问不同内存区域的速度不同的架构。文档中L3 Buffer以类NUMA方式分布在4×4 Mesh中。

**HBM（高带宽存储器，High Bandwidth Memory）**
 通过 TSV（硅通孔）将多层 DRAM 裸片垂直堆叠，再通过 2.5D 封装（如 CoWoS）与计算芯片紧密集成的内存。带宽可达 TB/s 级别（比 GDDR6 高数倍），是 AI GPU（如 H100/A100）的标配存储。

**Register File（寄存器堆）**
 处理器/加速器内部最快的存储单元，直接连接运算电路，容量极小（KB 级），零等待访问。PE 内部的 FP32 累加器就是寄存器的一种。

**Scratchpad Memory（暂存存储器/便签存储器）**
 由软件/编译器显式管理的片上 SRAM。与 Cache 不同：Cache 由硬件自动决定缓存什么数据，Scratchpad 由程序员（或编译器）精确控制什么数据搬进来、什么时候搬出去。你们芯片的 WL1/AL1/OL1 就是 Scratchpad 风格。

**Cache Coherence（缓存一致性）**
 多核系统中，各核心各自的缓存可能存了同一份数据的不同副本。缓存一致性协议（如 MESI）确保所有核心看到的数据始终是一致的。你们芯片的 L2 是核内私有的，不涉及 coherence；DRAM 是共享的，通过 SI/SO 指令手动同步。

### 🔷 互联与总线相关

**NoC（Network on Chip，片上网络）**
 芯片内部用于连接各模块的通信网络，类比"芯片内部的互联网"。

**2D-Mesh（二维网格拓扑）**
 NoC的一种拓扑结构，所有节点排成二维网格，每个节点与上下左右四个邻居相连。文档中NPU核心按4×4的2D-Mesh排列。

**AXI（Advanced eXtensible Interface）**
 ARM定义的高性能片上总线协议（AMBA协议族之一），广泛用于SoC设计，支持读写并发、乱序传输。

**FlexNoc（Arteris FlexNoc）**
 Arteris公司的商业片上网络IP，文档中用于SoC系统总线互联（外购）。

**Innolink**
 文档中使用的Die-to-Die高速互联接口，用于连接两个芯片裸片（实现Chiplet方案）。

**D2D（Die-to-Die）**
 裸片间直接互联接口，Chiplet方案的核心技术。

**PCIe（Peripheral Component Interconnect Express）**
 高速串行计算机扩展总线标准，常用于连接GPU、SSD等高速设备。文档中支持PCIe 4.0 x16，带宽256Gbps。RC指根复合体（主控端），EP指端点（从设备端）。

**RGMII（Reduced Gigabit Media Independent Interface）**
 千兆以太网的物理层接口标准。

**MIPI（Mobile Industry Processor Interface）**
 移动设备行业标准接口联盟定义的接口协议，常用于摄像头（MIPI CSI）和显示（MIPI DSI）连接。

**USB（Universal Serial Bus）**
 通用串行总线，文档支持USB 3.1（10Gbps）。

**I2C / SPI / UART / I2S / GPIO**
 常见低速接口：

- **I2C**：两线制串行总线，用于连接传感器等低速设备
- **SPI**：四线制高速串行接口
- **UART**：通用异步收发传输器，用于串口通信
- **I2S**：用于音频数据传输的串行总线
- **GPIO**：通用输入输出引脚

**EMMC（Embedded Multi Media Card）**
 嵌入式存储芯片，类似手机内置存储。

**QSPI Flash**
 四路SPI Flash存储器，用于存储启动引导程序（Bootloader）。

**SD 6.0**
 SD存储卡协议的第六代标准，支持更高传输速率。

**CXL（Compute Express Link）**
 新一代高速互联协议，基于 PCIe 物理层，支持处理器与加速器/内存扩展设备之间的缓存一致性互连。让 CPU 可以像访问本地内存一样访问远端内存。

**UCIe（Universal Chiplet Interconnect Express）**
 Chiplet（小芯片）之间互连的开放行业标准，定义了物理层和协议层规范。类比 PCIe 之于分立芯片，UCIe 之于同一封装内的多个小芯片。

**SerDes（串行器/解串器，Serializer/Deserializer）**
 高速接口的核心模块：发送端将并行数据转为高速串行信号（Serializer），接收端再还原为并行数据（Deserializer）。PCIe、以太网、USB 等高速接口都以 SerDes 为基础。

**PHY（物理层，Physical Layer）**
 接口协议的最底层硬件实现，负责电信号的发送/接收、时钟恢复、信号均衡等。芯片公司经常"买 PHY IP"——DDR PHY、PCIe PHY 指的就是这一层的硬件设计。

### 🔷 处理器与子系统相关

**Host CPU（主机CPU）**
 SoC中负责运行操作系统、调度任务的主处理器。文档中使用16核ARM A76，频率1.8~2.1GHz。

**ARM A76（ARM Cortex-A76）**
 ARM的高性能CPU核心设计（用于授权给芯片厂商使用），常见于2019-2020年旗舰手机。

**MCU（Micro-Controller Unit，微控制器）**
 低功耗小型控制器，文档中使用ARM China（安谋中国）的"星辰"系列，负责系统控制和低功耗管理（如休眠唤醒）。

**ARM China 星辰系列**
 安谋科技（ARM中国合资公司）自主研发的MCU IP产品线。

**ISP（Image Signal Processor，图像信号处理器）**
 负责将摄像头原始 RAW 数据转换为可用图像的专用处理器（去噪、白平衡、HDR 等）。**启明 940 本身不集成 ISP，依赖外部芯片**（935 的 ISP 集成情况请以项目 spec 为准）。

**CODEC（编解码器）**
 视频/图像的压缩编码和解压解码模块。文档中使用ARM LINLON-V5 IP，支持4K@240fps H.264/265解码。

**H.264 / H.265（HEVC）**
 视频压缩编码标准，H.265压缩率约是H.264的两倍。

**ARM LINLON-V5**
 ARM授权的视频编解码IP。

**DMA（Direct Memory Access，直接内存访问）**
 让外设直接读写内存，不需要CPU参与，释放CPU算力。文档中LOAD DMA和STORE DMA分别负责NPU的数据搬入和搬出。

**WDT（Watchdog Timer，看门狗定时器）**
 硬件计时器，若系统软件崩溃未能定时"喂狗"，则自动复位系统。

### 🔷 量化与神经网络相关

**量化（Quantization）**
 将神经网络中的浮点数（FP32）压缩为低精度整数（INT8/INT4），大幅降低存储和计算开销，代价是少量精度损失。

**PTQ（Post-Training Quantization，训练后量化）**
 训练完成后直接对模型进行量化，无需重新训练，操作简便但精度损失较训练感知量化稍大。

**非对称量化（Asymmetric Quantization）**
 用 `scale`（缩放因子）和 `zero_point`（零点偏移）两个参数做量化，能更好适应数据分布不均匀的情况（如ReLU后的激活值全为正数）。

**对称量化（Symmetric Quantization）**
 只用 `scale` 参数量化，零点固定为0，实现简单但对非对称分布精度稍差。

**通道级量化（Channel-wise Quantization）**
 为每个输出通道单独设置一组量化参数，精度高。文档中权重（Weight）采用此方式。

**张量级量化（Tensor-wise Quantization）**
 整个张量共用一组量化参数，参数少、实现简单。文档中激活值（Activation）采用此方式。

**scale（缩放因子）/ zero_point（零点）**
 非对称量化的两个核心参数，用于在浮点域和整数域之间做映射。

**Activation（激活值）**
 神经网络中层与层之间传递的特征图数据，是随输入动态变化的。

**Weight（权重）**
 神经网络中训练固定下来的参数，推理时不变。

**BN（Batch Normalization，批归一化）**
 训练中常用的归一化技术，将每层输出归一化到标准分布，加速训练、提升泛化。公式形式为 `Y = K·X + B`，推理时可与卷积层融合。

**反量化（Dequantization）**
 将整数计算结果还原回近似的浮点值，通常在BN之前执行。

**层融合（Layer Fusion）**
 将多个连续的算子（如Conv + BN + ReLU）合并为一个硬件操作，减少中间数据读写，大幅提升效率。

### 🔷 神经网络算子相关

**Conv2d（二维卷积）**
 最核心的神经网络算子，用滑动卷积核提取特征。

**Depthwise Conv（深度可分离卷积）**
 MobileNet的核心技术，将标准卷积拆分为深度卷积+逐点卷积，大幅减少计算量。

**Group Conv（组卷积）**
 将输入输出通道分成若干组，每组独立做卷积，减少参数量。AlexNet最早使用。

**Dilated Conv / Atrous Conv（空洞卷积）**
 在卷积核元素间插入空洞，扩大感受野而不增加参数，常用于语义分割。

**ConvTranspose（转置卷积/反卷积）**
 上采样操作，常用于语义分割和生成模型的解码器部分。

**Conv3d（三维卷积）**
 处理视频等三维数据的卷积，增加了时间维度。

**Linear / InnerProduct / FC（全连接层）**
 每个输入节点与每个输出节点都相连，常用于分类头部。

**Embedding（词嵌入层）**
 将离散词汇映射为连续向量的层，NLP模型的基础组件。

**ReLU / LeakyReLU / PReLU / ReLU-X**
 激活函数系列：

- **ReLU**：`max(0, x)`，最简单的激活函数
- **LeakyReLU**：负数部分有微小斜率，避免"神经元死亡"
- **PReLU**：斜率可学习的LeakyReLU
- **ReLU-X（ReLU6）**：上限裁剪为X（通常为6），适合量化

**Sigmoid / Tanh / Erf / GELU / GEGLU**
 其他激活函数：

- **Sigmoid**：输出(0,1)区间，用于二分类或门控
- **Tanh**：输出(-1,1)，用于RNN
- **Erf**：误差函数
- **GELU**：Gaussian Error Linear Unit，Transformer中常用
- **GEGLU**：GELU的门控变体，用于部分大语言模型

**Softmax**
 将向量归一化为概率分布，常用于分类输出层。

**Maxpool / AvgPool / AdaptiveAvgPool（池化层）**
 通过取最大值/平均值降低特征图尺寸，减少计算量。

**BatchNorm / LayerNorm（归一化层）**

- **BatchNorm**：对一批样本的同一通道做归一化
- **LayerNorm**：对单个样本的所有通道做归一化，Transformer常用

**Upsample（上采样）**
 增大特征图尺寸，常用模式有最近邻（Nearest）和双线性插值（Bilinear）。

**Element-wise Add/Mul（逐元素加/乘）**
 对两个形状相同的张量逐元素操作，如ResNet的残差连接。

**Permute / Transpose / Reshape / Flatten / Concat / Split / Squeeze / Unsqueeze**
 维度变换类算子，不涉及计算，主要改变数据的存储排列方式，通常由编译器处理。

**Gather / Scatter**
 按索引从张量中取元素/写元素，常用于Embedding查表。

**Einsum（爱因斯坦求和约定）**
 用简洁符号描述复杂张量运算（如矩阵乘、批量矩阵乘等），Transformer中常见。

**Dropout**
 训练时随机丢弃神经元，防止过拟合。推理时无效，文档明确标注"Inference没有该算子"。

### 🔷 指令架构相关

**指令集（ISA，Instruction Set Architecture）**
 定义处理器能执行的操作的规范。文档要求"后向兼容"，即新一代芯片的指令集能运行为老芯片写的程序。

**乱序发射（Out-of-Order Issue）**
 不按程序顺序发射指令，充分利用硬件并行性。文档中通过"信箱（Mailbox）计数器"机制管理指令间依赖。

**收信位 / 寄信位**
 **Zeus V2（对应启明 935）** 自定义的指令依赖表示方式：收信位表示"我需要等谁完成"，寄信位表示"我完成后要通知谁"。这是 935 汇编器生成二进制指令时要处理的核心机制之一。

**指令类型（LD/ST/LW/SI/SO/DT/PE/VP/SYNC）**

- **LD**：从DDR加载数据到L2
- **LW**：从DDR或L2加载权重到WL1
- **ST**：从L2存储数据到DDR
- **SI（Sync In）/ SO（Sync Out）**：核间同步与数据传输
- **DT（Data Transform）**：维度变换
- **PE**：卷积/FC计算
- **VP（Vector Processor）**：池化等向量运算

### 🔷 数据流与映射相关

**WS（Weight Stationary，权重驻留）**
 数据流策略：将权重固定在计算单元的本地存储中不动，不断流入不同的激活值进行复用。减少权重的重复读取。

**Partition（分割/映射）**
 将神经网络层的计算任务分配到多个PE/Core上并行执行的方式。文档中定义了9种Partition模式（WH、WC、NH、NC、KH、KC等）。

**Layer Pipeline（层流水线）**
 多个网络层同时在不同核上并行流水执行，提升吞吐量。适用于小网络且权重能全部放在片上存储的情况。

**NHWC / NCHW**
 神经网络中Tensor的维度排列方式：N=Batch，H=Height，W=Width，C=Channel。不同框架默认不同，对硬件访存效率影响很大。

**Tile（数据块）**
 将大规模数据分块处理的单位，是AI芯片编译器做内存优化的核心概念。

**Padding（填充）**
 在输入特征图边缘填充零值，保持卷积输出尺寸或满足算法需求。

**Stride（步长）**
 卷积核滑动的步幅，影响输出特征图大小。

### 🔷 半导体物理与晶体管

**半导体（Semiconductor）**
 导电性介于导体（铜）和绝缘体（橡胶）之间的材料，代表是硅（Si）。可通过掺杂精确控制导电性——这就是造芯片的物理基础。

**晶体管（Transistor）**
 现代芯片的基本"开关"单元，通过控制信号控制电流通断。类比水管中的阀门——旋钮一转水就流。一颗现代 CPU 含数百亿颗晶体管。

**MOSFET（金属-氧化物-半导体场效应晶体管）**
 现代芯片最主流的晶体管类型。三个端子：**栅极（Gate）**——控制端，类比阀门旋钮；**源极（Source）**——电流入口；**漏极（Drain）**——电流出口。栅极电压控制源漏之间的沟道是否导通。

**NMOS / PMOS**
 两种互补的 MOSFET。NMOS 加正电压导通（电子导电，速度快）；PMOS 加负电压导通（空穴导电，速度慢）。两者配合使用就是 CMOS。

**CMOS（互补型 MOS，Complementary MOS）**
 将 NMOS 和 PMOS 配对使用的电路技术，是现代数字芯片的基石。最大优点：静态时几乎不耗电（只在开关瞬间消耗功率），极其省电。

**沟道（Channel）**
 源极和漏极之间、受栅极控制的导电区域。沟道长度（Channel Length）就是"工艺节点"最早的物理含义——越短，开关越快，芯片越先进。

**阈值电压（Threshold Voltage，Vth）**
 使晶体管刚好开始导通的最低栅极电压。Vgs > Vth 时晶体管打开，Vgs < Vth 时关闭（但关不死，见漏电流）。

**漏电流（Leakage Current）**
 即使晶体管"关闭"（Vgs < Vth），仍有微量电流偷偷流过沟道（亚阈值漏电）。工艺越先进、沟道越短，漏电越严重——这是芯片越来越难省电的根本原因。

**短沟道效应（Short Channel Effect，SCE）**
 沟道长度缩短后，漏极电场对沟道的影响增强，导致阈值电压不稳、漏电增大、栅极控制力下降等一系列问题。是推动晶体管从平面走向 3D 结构（FinFET/GAA）的根本原因。

**FinFET（鳍式场效应晶体管，Fin Field-Effect Transistor）**
 将沟道做成垂直的"鳍片"状，栅极从三面（上和两侧）包裹沟道，控制力大幅提升。Intel 在 22nm 首次商用，之后成为行业标准（一直用到 ~5nm）。类比：平面 MOSFET 是"用手掌盖住水管"，FinFET 是"用三根手指捏住水管"——控制更精准。

**GAA（全环绕栅极，Gate-All-Around）**
 下一代晶体管结构：栅极从**四面**完全包裹沟道（vs FinFET 的三面），控制力最强。沟道做成水平堆叠的纳米薄片（Nanosheet），三星 3nm 和台积电 2nm 开始采用。类比：FinFET 是"捏住水管"，GAA 是"套了个管箍"。

**Nanosheet（纳米薄片）**
 GAA 的主流实现形式——多层水平硅薄片堆叠，每层都被栅极包裹。通过调整薄片宽度灵活调节驱动电流。

**HKMG（高 k 金属栅，High-k Metal Gate）**
 将传统 SiO₂ 栅介质替换为高介电常数材料（如 HfO₂），多晶硅栅极替换为金属栅。45nm 节点起成为标准，解决了栅极缩小到极限时的漏电问题。

**PN 结（P-N Junction）**
 P 型和 N 型半导体接触处形成的结，具有单向导电性。是二极管、晶体管等几乎所有半导体器件的基础结构。

### 🔷 芯片制造工艺（从沙子到芯片）

> 芯片制造的核心流程：在晶圆上反复执行"光刻 → 刻蚀/沉积/掺杂 → CMP"的循环，逐层堆叠出晶体管和互连线。一颗先进芯片需要经历几百道工序、数周时间。

**晶圆（Wafer）**
 芯片的"画布"。由高纯度单晶硅拉成硅棒，再切成薄圆片。直径常见 200mm（8 英寸）或 300mm（12 英寸），越大的晶圆能切出越多芯片。所有电路都在这张圆片上制造，最后才切开。

**硅锭（Ingot）**
 用直拉法（CZ 法）从熔融硅中拉出的单晶硅棒。切片后就成了晶圆。纯度要求极高——99.9999999%（"九个九"）。

**光刻（Photolithography / Lithography）**
 芯片制造**最核心**的工序。用光把掩模版上的电路图案"投影"到涂了光刻胶的晶圆上，像照相一样把图案转印到硅片表面。整个芯片制造围绕光刻展开——先印图案，再根据图案去做刻蚀或沉积。

**掩模版（Mask / Reticle，光罩）**
 印有电路图案的"底片/模板"。光通过掩模版上的透明/不透明区域投射到晶圆上。一颗芯片需要几十张掩模版（每层一张），一套 7nm 掩模版费用可达数千万美元。

**光刻胶（Photoresist，PR）**
 涂在晶圆表面的感光材料。曝光后化学性质改变：正胶——曝光区变得可溶（冲掉曝光区）；负胶——曝光区变得不可溶（保留曝光区）。

**曝光（Exposure）**
 光刻机用光源（DUV 或 EUV）照射掩模版，将图案投影到光刻胶上的过程。

**显影（Development）**
 曝光后用化学溶液冲洗掉可溶区域的光刻胶，留下图案轮廓。类比洗照片——曝光是拍照，显影是冲印。

**DUV（深紫外光刻，Deep Ultraviolet）**
 使用 193nm（ArF）波长光源的光刻技术，是目前应用最广泛的方案。配合浸没式光刻和多重图形技术，可以做到约 7nm 节点。

**EUV（极紫外光刻，Extreme Ultraviolet）**
 使用 13.5nm 波长的光刻技术，7nm 以下先进工艺的核心装备。全球仅 ASML 一家能造，每台约 1.5 亿美元。波长短 = 能印更细的线。

**浸没式光刻（Immersion Lithography）**
 在透镜和晶圆之间注入超纯水代替空气，利用水的折射率（1.44）提高分辨率。ArF 浸没式（ArFi）是 DUV 时代的主力。

**多重图形技术（Multi-Patterning）**
 在 EUV 普及前，靠一台光刻机分辨率不够怎么办？答案是同一层用多次曝光+刻蚀拼出更细的图案。LELE（Litho-Etch-Litho-Etch）、SADP/SAQP 是常见方案。成本高，但用 DUV 硬做到了 7nm。

**OPC（光学临近修正，Optical Proximity Correction）**
 光在极小尺度下会衍射，导致印出的图案和掩模版上的不一样（角变圆、线变粗等）。OPC 预先在掩模版图案上做反向补偿——把角故意做尖、线故意做细——让实际印出来的刚好是想要的形状。

**刻蚀（Etching）**
 光刻后，把没有光刻胶保护的材料"啃掉"的工序。类比用酸腐蚀金属——但精度是纳米级的。

**干法刻蚀（Dry Etching）**
 用等离子体（电离气体）轰击材料表面，物理+化学双重作用去除材料。优点：方向性好（只往下刻，不往旁边刻），先进节点几乎全用干刻。

**湿法刻蚀（Wet Etching）**
 用化学溶液腐蚀材料。各方向都刻（各向同性），精度不如干刻，但便宜，多用于清洗或不要求精度的工序。

**沉积（Deposition）**
 在晶圆表面"长"出一层新材料的工艺，类比刷漆。可以沉积绝缘层、金属层、半导体层。

**CVD（化学气相沉积，Chemical Vapor Deposition）**
 通入反应气体，在高温或等离子体辅助下发生化学反应，在晶圆表面生成固态薄膜。覆盖性好，最常用的沉积方法。

**PVD（物理气相沉积，Physical Vapor Deposition）**
 用物理方法（溅射）将靶材原子打出来沉积到晶圆上。常用于沉积金属层（铝、钛等）。

**ALD（原子层沉积，Atomic Layer Deposition）**
 每次只沉积一个原子层厚度，通过交替通入不同气体实现。精度极高，是先进节点高 k 栅介质的核心工艺。

**掺杂（Doping）**
 向纯硅中引入杂质原子，改变其导电类型。掺磷/砷 → N 型（多电子）；掺硼 → P 型（多空穴）。类比往纯水里加盐使其导电。

**离子注入（Ion Implantation）**
 最主流的掺杂方式：将杂质原子电离后用高压电场加速，直接轰入硅中。精度高，可精确控制掺杂深度和浓度。

**退火（Annealing）**
 离子注入后对晶圆进行高温处理，修复注入造成的晶格损伤，同时激活杂质原子。快速热退火（RTA）可在数秒内完成。

**热氧化（Thermal Oxidation）**
 将硅片放在高温氧气中，表面硅与氧反应生成 SiO₂ 绝缘层。是最基础的绝缘层制备方法。

**CMP（化学机械抛光，Chemical Mechanical Planarization）**
 兼用化学腐蚀和机械研磨将晶圆表面磨平。多层沉积后表面凹凸不平，CMP 后才能进行下一层光刻（平整度要到纳米级）。类比用砂纸把毛坯墙磨光滑。

**FEOL（前道工序，Front-End of Line）**
 晶圆制造的前半段：制造晶体管本身（源极/漏极/栅极）。

**BEOL（后道工序，Back-End of Line）**
 晶圆制造的后半段：制造金属互连线和通孔，把晶体管"接线"成完整电路。先进芯片有十几到二十层金属互连。

**MOL（中道工序，Middle of Line）**
 FEOL 和 BEOL 之间的过渡层，制造接触孔（Contact）——把晶体管端子引出给互连层。

**金属互连（Metal Interconnect）**
 BEOL 中连接不同晶体管的金属导线。现代芯片用铜（Cu），从 M1（最底层）到 Mx（最顶层），层数越多布线越灵活。

**通孔（Via）**
 连接不同金属层的垂直导线通道。类比楼层间的电梯井——每层金属线是一层楼，Via 是楼层之间的连接。

**双镶嵌工艺（Dual Damascene）**
 铜互连的制造方式：先在绝缘层中刻出沟槽和通孔形状，再填铜，最后 CMP 磨掉多余的铜。与传统"先沉积金属再刻蚀"相反。

**Low-k 介质（低介电常数材料）**
 金属互连层之间的绝缘材料。k 值越低，线间寄生电容越小，信号越快、功耗越低。先进节点 Low-k 材料的 k 值可低至 2.0 以下。

**STI（浅槽隔离，Shallow Trench Isolation）**
 在相邻晶体管之间刻沟填绝缘体，防止漏电串扰。类比在两块田之间挖排水沟。

**洁净室（Clean Room）**
 芯片制造必须在超高洁净度环境下进行。等级用 ISO 标准衡量——光刻区域通常是 ISO 3 级，每立方米空气中 ≥0.1μm 的颗粒不超过 1000 个（普通室外空气含数百万颗粒）。一粒灰尘就能毁掉一颗芯片。

**工艺节点（Process Node）**
 描述芯片制造工艺世代的标签，如 7nm、5nm、3nm。**注意**：现代节点的数字已不直接对应任何物理尺寸，更多是代际标签/营销用语。台积电的"3nm"和三星的"3nm"实际晶体管尺寸也不同。

**缺陷密度（Defect Density，D₀）**
 每平方厘米晶圆上的平均缺陷数。与良率直接挂钩：缺陷密度越低 → 良率越高 → 成本越低。

**PDK（工艺设计套件，Process Design Kit）**
 晶圆厂给芯片设计公司的"制造说明书"：包含器件参数模型、设计规则、标准单元库等。Fabless 公司拿到 PDK 才能针对这个工艺做设计。

### 🔷 芯片封装技术

> 芯片制造完成后还是一颗裸片（Die），不能直接用。封装的作用：①机械保护 ②散热 ③把 Die 上密密麻麻的微小焊盘引出成 PCB 板能焊接的引脚。

**封装（Packaging）**
 将裸片（Die）保护起来、引出电气接口、使之成为可以焊到电路板上的成品芯片的全过程。

**焊盘（Bond Pad / Pad）**
 裸片表面的金属方块，是芯片与外部连接的"接线柱"。先进芯片上有数千个焊盘。

**凸点（Bump）**
 焊盘上制作的金属小球或柱，用于 Flip Chip 封装中芯片与基板的直接连接。

**引线键合（Wire Bonding）**
 最经典的封装互连方式：用极细的金/铜/铝线将芯片焊盘与引线框架或基板引脚连接。成本低，但引线有寄生电感，不适合高频高速。

**Flip Chip（倒装芯片）**
 将芯片翻转 180°，焊盘朝下通过凸点（Bump）直接连到基板上，不需要引线键合。互连距离短、寄生效应小，是高性能芯片（CPU/GPU/NPU）的主流封装方式。

**引线框架（Lead Frame）**
 最传统的封装支撑结构——冲压出的金属框架，芯片贴在中间，外围引脚连 PCB。便宜但引脚数有限。

**封装基板（Package Substrate / IC 载板）**
 高密度封装中代替引线框架的多层布线板。比 PCB 布线更密，作为芯片到 PCB 之间的"转接板"。BGA 封装必须有基板。

**DIP（双列直插封装，Dual In-line Package）**
 最古老的封装之一，两排引脚直插 PCB 孔。现在主要见于教学用 IC 和分立器件。

**QFP（四侧引脚扁平封装，Quad Flat Package）**
 四面都有引脚向外伸出的扁平封装。早期 MCU、FPGA 常见。

**QFN（四侧无引脚扁平封装，Quad Flat No-Lead）**
 类似 QFP 但引脚不伸出来，藏在封装底面。更小更薄，散热更好。

**BGA（球栅阵列封装，Ball Grid Array）**
 封装底面排布锡球阵列作为引脚，引脚数可以远多于四边引脚封装。是现代高引脚数芯片（CPU/GPU/FPGA）的主流封装形式。

**CSP（芯片尺寸封装，Chip Scale Package）**
 封装后尺寸接近裸片本身（不超过芯片面积 1.2 倍），极大减小占板面积。

**WLP（晶圆级封装，Wafer Level Package）**
 在晶圆切割之前就完成封装互连，切割后即为成品。尺寸最小，成本可摊薄。

**Fan-Out（扇出封装）**
 晶圆级封装的升级版：将芯片嵌入模塑料重构成更大面积的"假晶圆"，布线区域可以"扇出"到芯片边界之外，I/O 数不受芯片面积限制。苹果 A 系列处理器用台积电 InFO（扇出封装）。

**RDL（重布线层，Redistribution Layer）**
 晶圆级封装中新增的金属布线层，将芯片原始焊盘位置重新分布引出，类比"延长线"。

**Underfill（底部填充胶）**
 Flip Chip / BGA 中，在芯片和基板之间填入环氧树脂胶。防止热膨胀系数不匹配导致凸点断裂。

**Molding（塑封/模封）**
 用环氧模塑料（EMC）将芯片和互连结构整体包裹，提供机械保护和防潮。

**2.5D 封装**
 将多个裸片并排放在一块中介层（Interposer）上，通过中介层上的高密度互连实现芯片间超近距离通信。称"2.5D"是因为芯片水平排列但没有完全垂直堆叠。

**3D 封装 / 3D-IC**
 将多个芯片垂直堆叠，用 TSV 在 Z 方向互连。HBM 内存就是典型的 3D 封装（多层 DRAM Die 堆叠）。

**Interposer（中介层/硅转接板）**
 介于芯片和封装基板之间的一层"桥"，上面有极高密度的互连布线。硅中介层用光刻工艺制作，布线密度接近芯片内部。

**TSV（硅通孔，Through-Silicon Via）**
 贯穿硅片的垂直导电通道——在硅片上钻孔并填充铜。是 3D 堆叠和中介层互连的关键技术。类比在楼板上打洞穿电线。

**CoWoS（Chip on Wafer on Substrate）**
 台积电的旗舰 2.5D 封装方案。NVIDIA H100/H200 用它把 GPU Die 和 HBM 堆叠在同一个硅中介层上。目前 AI 芯片最主流的先进封装。

**InFO（Integrated Fan-Out）**
 台积电的扇出封装技术，没有基板，厚度极薄。苹果 iPhone 处理器使用此技术。

**EMIB（Embedded Multi-die Interconnect Bridge）**
 英特尔的 2.5D 方案：在封装基板中局部嵌入小硅桥，不需要完整硅中介层，成本低于 CoWoS。

**混合键合（Hybrid Bonding）**
 最先进的芯片堆叠互连方式：两颗芯片的铜焊盘直接面对面键合，无需凸点，互连密度最高。Sony CMOS 传感器和台积电 SoIC 均采用此技术。

**Dicing（晶圆切割）**
 将晶圆沿切割道切成单颗裸片，用金刚石锯片或激光完成。

### 🔷 芯片测试（DFT 与量产测试）

> 芯片造出来不能直接卖——必须经过严格测试筛选良品。测试贯穿从晶圆到成品全流程。

**CP 测试（Chip Probing / Wafer Sort，晶圆测试）**
 晶圆切割前，用探针直接接触每颗 Die 的焊盘做电学测试。目的：找出坏 Die 并标记，避免浪费封装成本。

**FT 测试（Final Test，终测）**
 封装完成后对成品芯片做最终全项测试。测试环境更接近实际使用，并做最终分选（Binning）。

**SLT（System Level Test，系统级测试）**
 将芯片放在接近真实系统的环境下（配真实 DRAM、电源等）测试，发现功能测试覆盖不到的深层问题。

**ATE（自动测试设备，Automatic Test Equipment）**
 执行芯片测试的专用大型机器，由测试机、探针台/分选机和接口板组成。主流供应商：Advantest、Teradyne。

**探针台（Prober）**
 CP 测试中承载晶圆、使探针卡依次接触每颗 Die 的设备。

**探针卡（Probe Card）**
 CP 测试中与 Die 焊盘接触的精密接口板，上面有数千根极细探针。是高消耗品。

**分选机（Handler）**
 FT 测试中负责取放芯片、送入测试座、按结果分类收集的自动机械臂设备。

**DFT（可测性设计，Design for Testability）**
 在芯片设计阶段专门加入辅助测试的逻辑电路，使外部 ATE 能有效控制和观测芯片内部状态。没有 DFT，复杂芯片内部几乎无法测试。

**扫描链（Scan Chain）**
 DFT 的核心技术。将芯片内部的触发器改造成可串联的扫描单元，串成"项链"。测试时通过串行移位将测试数据送入芯片内部（Scan-in），读出响应（Scan-out），绕过正常逻辑直接检测故障。

**ATPG（自动测试向量生成，Automatic Test Pattern Generation）**
 EDA 工具根据电路网表自动生成能最大化覆盖故障的测试向量，配合扫描链使用。

**BIST（内建自测试，Built-In Self Test）**
 在芯片内部集成测试逻辑，让芯片能自我测试，减少对外部 ATE 的依赖。MBIST（Memory BIST）专测片上 SRAM/ROM；LBIST（Logic BIST）测逻辑电路。

**JTAG（边界扫描接口，Joint Test Action Group）**
 IEEE 1149.1 标准的串行测试/调试接口（4~5 根线：TCK/TMS/TDI/TDO/TRST）。几乎所有处理器和 FPGA 都有 JTAG，可用于芯片调试、Flash 烧录、板级互连测试。

**Burn-in（老化测试）**
 将芯片在高温（~125°C）、高电压下持续运转数小时到数天，提前筛除"先天不足"的早期失效品。类比汽车出厂前的高强度路测。

**Binning（分选/分 Bin）**
 测试后根据芯片实际性能（频率、功耗）分入不同等级。同一晶圆的芯片，跑得快的卖高价（i9），跑得慢的降级卖（i5），最大化利用晶圆价值。

**故障覆盖率（Fault Coverage）**
 测试向量能检测到的潜在故障比例。先进芯片要求 >99%。DFT 的核心目标就是提高覆盖率。

**DPM（每百万件缺陷数，Defects Per Million）**
 衡量出货质量的指标。车规芯片要求 <1 DPM（百万颗中不到 1 颗有缺陷）。

### 🔷 EDA 工具与芯片设计流程

> 没有 EDA 软件，现代芯片无法设计。EDA 之于芯片 = IDE 之于软件开发。三大供应商：Synopsys、Cadence、Siemens EDA。

**EDA（电子设计自动化，Electronic Design Automation）**
 芯片设计使用的全套软件工具，涵盖逻辑设计、综合、布局布线、验证、DFT 等。

**RTL（寄存器传输级，Register Transfer Level）**
 描述芯片逻辑功能的抽象层次：数据从寄存器经过组合逻辑流向另一组寄存器。是芯片前端设计的核心产出物。类比软件中的源代码。

**Verilog / SystemVerilog / VHDL（硬件描述语言，HDL）**
 编写 RTL 的语言，类比 C/Python 之于软件。Verilog 最常用；SystemVerilog 是其增强版，还有验证语法；VHDL 在欧洲和军工常见。

**综合（Synthesis）**
 把 RTL 代码"编译"成门级网表（具体的与门、或门、触发器连接）的过程。类比高级语言编译成机器码。常用工具：Synopsys Design Compiler。

**网表（Netlist）**
 综合后的输出——描述芯片中每个逻辑门及其连线的文本清单。是前端设计和后端物理设计之间的"施工图纸"。

**标准单元库（Standard Cell Library）**
 包含与门、或门、触发器等基本单元的物理版图和时序参数的集合。综合工具从这里"选材料"来实现网表。

**Floorplan（平面规划）**
 在芯片版图上划定各模块的位置和面积，规划电源网络和 I/O 引脚。类比盖楼前的平面图。

**Placement（布局）**
 把综合后的标准单元放到芯片上的合理位置，目标是线短、时序好、不拥堵。

**CTS（时钟树综合，Clock Tree Synthesis）**
 芯片内所有触发器都需要同一个时钟信号。CTS 通过插入缓冲器让时钟均匀地到达每个触发器——避免有的地方快有的地方慢。类比给大楼的每间教室安装同步铃声。

**Routing（布线）**
 把布局好的单元之间按网表要求用金属导线连接起来。是整个物理设计最耗时的步骤。

**P&R / PnR（布局布线，Place and Route）**
 Placement + Routing 的合称，后端物理设计的核心环节。常用工具：Cadence Innovus、Synopsys ICC2。

**DRC（设计规则检查，Design Rule Check）**
 检查版图是否满足晶圆厂的制造规则（最小线宽、间距等）。DRC 不通过就不能流片——类比施工图审批不通过就不能开工。

**LVS（版图与原理图对比，Layout vs. Schematic）**
 对比最终版图和原始设计网表是否一致。通不过说明"盖出来的楼和设计图不一样"。

**STA（静态时序分析，Static Timing Analysis）**
 不运行仿真，纯靠数学计算分析所有信号路径延迟，检查是否满足频率要求。常用工具：Synopsys PrimeTime。

**Timing Closure（时序收敛）**
 通过反复迭代优化，使所有路径时序都满足要求的过程。是后端设计中最难、最耗时的部分。

**Sign-off（签核）**
 流片前确认设计满足所有规格（DRC/LVS 清零、STA 收敛、功耗 OK、IR Drop OK）的正式决策节点。过了签核才能提交给晶圆厂。

**GDS / GDSII（版图数据格式）**
 芯片最终版图的标准文件格式，包含所有层的几何形状数据。是交给晶圆厂的"最终施工图"。

**ECO（工程变更单，Engineering Change Order）**
 设计快完成甚至流片后发现小 bug，做局部小修改而不推倒重来。类比施工快结束时发现少开了个窗，局部动工。

**IP Core（知识产权核，Intellectual Property Core）**
 预先设计好的可复用功能模块，像积木一样集成到芯片中。例如 USB 控制器、DDR 接口、PCIe PHY。分硬核（Hard IP，版图已定，不可改但性能最优）和软核（Soft IP，只有 RTL，灵活但需自己跑后端）。

**PPA（性能/功耗/面积，Performance / Power / Area）**
 芯片设计的三大核心优化目标，三者互相制约。跑得快、还省电、还小——不可能全部做到最优，所以芯片设计本质是在三者之间做 trade-off。

**DFM（可制造性设计，Design for Manufacturability）**
 设计阶段就考虑制造约束，通过加宽导线、冗余过孔等手段提高良率。

### 🔷 芯片验证

> 验证 = 确认芯片设计是否正确。占芯片开发 60%~70% 的工时——写代码容易，证明代码没 bug 难。

**仿真（Simulation）**
 用软件运行 RTL 代码，输入各种激励，观察输出是否符合预期。最基础、最常用的验证手段。缺点：慢——复杂系统跑一个场景可能要数小时。

**Testbench（测试平台）**
 包裹在被测设计（DUT）外面的验证框架，负责产生输入、监控输出、判断对错。类比：芯片是考生，testbench 是出题老师+阅卷机。

**UVM（通用验证方法学，Universal Verification Methodology）**
 基于 SystemVerilog 的标准化验证框架，提供可复用的组件（Driver、Monitor、Scoreboard 等）。是工业界芯片验证的事实标准。

**覆盖率（Coverage）**
 衡量验证完整性：代码覆盖率（哪些代码被执行到了）+ 功能覆盖率（哪些场景被测到了）。覆盖率达标是验证"收工"的重要依据。

**断言（Assertion / SVA）**
 在 RTL 中插入形式化规则，如"信号 A 拉高后 3 周期内 B 必须响应"。仿真时自动检查，违反则报错。

**回归测试（Regression）**
 每次改代码后，重跑全部测试用例确保没引入新 bug。类比软件开发的 CI/CD。

**Emulation（硬件仿真）**
 将 RTL 下载到专用大型硬件仿真机上运行，速度比软件仿真快 100~1000 倍，可以跑操作系统和真实应用。设备极贵。常用：Cadence Palladium、Synopsys ZeBu。

**FPGA 原型验证（FPGA Prototyping）**
 把 RTL 综合后下载到商业 FPGA 板上运行，速度接近真实时钟频率。比 emulation 便宜，但 FPGA 容量有限。常用于软硬件协同开发。

**形式化验证（Formal Verification）**
 用数学方法证明设计正确，不依赖测试激励。分等价性检查（网表是否等价于 RTL）和属性检查（断言是否在所有状态下都成立）。

**SystemC / TLM（事务级建模）**
 用 C++ 写芯片行为模型，以"事务"（如一次读请求）而非信号波形描述通信。仿真极快，用于早期架构探索和虚拟原型（在 RTL 未完成前就跑嵌入式软件）。

**工艺角（Process Corner）**
 表征工艺偏差的极端参数组合：TT（典型-典型）、FF（快-快）、SS（慢-慢）、FS、SF。芯片需在所有工艺角 + 温度/电压极端组合下都满足时序——这就是 PVT（Process-Voltage-Temperature）分析。

### 🔷 功耗与散热

**动态功耗（Dynamic Power）**
 芯片工作时因晶体管开关翻转消耗的功率。公式：P = α·C·V²·f（翻转率×电容×电压²×频率）。降电压是最有效的降功耗手段（V² 关系）。

**静态功耗 / 漏电功耗（Static Power / Leakage Power）**
 芯片不干活也在白白消耗的功率，来自晶体管关不死的漏电流。工艺越先进（沟道越短），漏电越严重。先进节点静态功耗可占总功耗的 30%~40%。

**TDP（热设计功耗，Thermal Design Power）**
 芯片持续满负荷工作时的最大散热需求。买散热器/设计散热方案时看的就是这个数。

**DVFS（动态电压频率调节，Dynamic Voltage and Frequency Scaling）**
 根据负载动态调压调频：轻负载时降压降频省电，重负载时升压升频保性能。手机 SoC 无处不在。

**Power Gating（电源门控）**
 直接切断不需要工作的模块的电源——最彻底的省电手段，漏电功耗降为零。需要配合保留寄存器（Retention Register）保存状态。

**Clock Gating（时钟门控）**
 关闭不需要工作的模块的时钟信号——消除不必要的动态功耗。是最常用、收益最显著的低功耗技术。

**电源域（Power Domain）**
 芯片中共享同一组电源的电路区域。多电源域设计允许不同模块独立上电/断电/调压。

**Multi-Vt（多阈值电压）**
 同一芯片混合使用不同阈值电压的晶体管：HVT（高阈值，慢但漏电小）用于非关键路径；LVT（低阈值，快但漏电大）用于关键路径。在速度和漏电之间取最优平衡。

**IR Drop（电源压降）**
 电流流过电源网络金属线的电阻时产生压降（V=IR）。压降过大 → 某些区域供电不足 → 时序违例甚至功能错误。签核前必须分析。

**Thermal Throttling（热节流）**
 芯片温度超过安全阈值时自动降频降压，防止烧毁。表现为突然降速——笔记本玩游戏变卡很多时候就是在 throttling。

**隔离单元（Isolation Cell）**
 当一个电源域断电时，其输出信号变为不确定状态（X 态），可能损坏邻域。隔离单元插在域边界，将输出钳位到安全电平。

**电平转换器（Level Shifter）**
 两个电源域电压不同时（如 1.8V 和 0.9V），跨域信号需要通过电平转换器适配电压。

### 🔷 时钟与信号完整性

**PLL（锁相环，Phase-Locked Loop）**
 输入参考时钟（如 100 MHz），输出倍频/分频/调相后的精确时钟（如 1 GHz）。是芯片内部产生各种工作频率的核心模块。

**DLL（延迟锁定环，Delay-Locked Loop）**
 类似 PLL 但只做相位对齐，不做频率变换。常用于 DDR 内存接口的时钟校准。

**Clock Skew（时钟偏斜）**
 同一时钟信号到达芯片不同触发器的时间差。由走线长度差异引起。CTS 的目标就是最小化 skew。

**Clock Jitter（时钟抖动）**
 时钟周期围绕理想值的随机波动，由电源噪声、PLL 噪声等引起。与 skew（空间差异）不同，jitter 是同一点的时间差异。

**Setup Time（建立时间）**
 数据信号必须在时钟沿到来**之前**稳定保持的最短时间。违反 → 采样错误。

**Hold Time（保持时间）**
 数据信号在时钟沿到来**之后**还必须继续保持稳定的最短时间。违反 → 同样采样错误。

**Critical Path（关键路径）**
 所有信号路径中延迟最大的那条。它决定了芯片能跑的最高频率——优化时序就是缩短关键路径。

**Slack（时序裕量）**
 要求到达时间 - 实际到达时间。正 Slack = 满足要求；负 Slack = 时序违例，必须修。

**亚稳态（Metastability）**
 当触发器的建立/保持时间被违反（常见于异步信号采样），输出电平长时间悬停在 0 和 1 之间的不确定状态。是跨时钟域设计的核心危险。

**CDC（跨时钟域，Clock Domain Crossing）**
 信号从时钟 A 的电路传到时钟 B 的电路。两个时钟频率/相位无关，直接采样极易发生亚稳态。解决方案：同步器（串联 2~3 级触发器），或异步 FIFO（跨域传多位数据）。

**Crosstalk（串扰）**
 相邻信号线之间通过寄生电容/互感发生的干扰——一条线的信号变化会影响旁边的线。严重时导致时序偏移或功能错误。

**EMI（电磁干扰，Electromagnetic Interference）**
 电路工作时向外辐射电磁波干扰其他设备，或被外部电磁波干扰。高速芯片和 PCB 的重要设计约束。

### 🔷 芯片行业与产业链

**Fabless（无晶圆厂设计公司）**
 只做芯片设计，不建晶圆厂，制造外包给代工厂。代表：高通、英伟达、AMD、海思、联发科。你们公司大概率也是 Fabless 模式。

**Foundry（晶圆代工厂）**
 专门承接芯片制造的纯制造商。代表：台积电（TSMC）、中芯国际（SMIC）、格芯（GlobalFoundries）。

**IDM（集成器件制造商，Integrated Device Manufacturer）**
 设计+制造+封测全自营。代表：Intel、三星、德州仪器。资产最重，但工艺与设计协同最好。

**OSAT（封测代工厂，Outsourced Semiconductor Assembly and Test）**
 专做封装和测试的外包企业。代表：日月光（ASE）、长电科技。产业链"后道"环节。

**IP Vendor（IP 供应商）**
 专门设计和授权可复用 IP 核的公司。代表：ARM（CPU 核）、Synopsys（接口 IP）、Cadence（SerDes IP）。

**NRE（一次性工程费用，Non-Recurring Engineering）**
 芯片研发和流片的一次性投入——掩模版费、设计工具费、人力费等。不随产量摊薄。先进工艺（5nm）NRE 可达数千万美元。

**BOM（物料清单，Bill of Materials）**
 产品所有组成材料的清单，是估算成本的基础。

**Design Win（设计导入）**
 芯片被终端客户（如车厂）正式选定并纳入其产品设计的里程碑。Design Win ≠ 出货，但意味着进入客户供应链。

**License（授权费）**
 使用某项 IP/技术的一次性许可费。如 ARM 架构授权给芯片公司。

**Royalty（版税/提成费）**
 按每颗芯片出货量支付给 IP 方的费用，通常按芯片售价的百分比计算。

**Lead Time（交期）**
 从下单到交货的时间。芯片交期通常 12~26 周，缺芯时可达半年以上。

**ES / QS / CS / MP（产品阶段）**
 ES（Engineering Sample，工程样品）= 流片后第一批芯片，内部验证用；QS（Qualified Sample，认证样品）= 送客户/机构做认证；CS（Commercial Sample，商用样品）= 确认量产条件；MP（Mass Production，量产）= 正式大规模出货。

**First Silicon（首片硅）**
 流片后回来的第一批物理芯片。验证 First Silicon 是否能工作是流片后最紧张的一关。

**Silicon Proven（硅验证通过）**
 某个 IP 已经在真实芯片上测试确认功能正确。比"仅仿真验证"可信度高得多。

**Shuttle Run（穿梭验证/MPW 班车）**
 晶圆厂定期组织的 MPW 机会，多个客户拼一片晶圆分摊掩模费。设计团队赶在截止日前完成 Tapeout 即可搭"班车"。

### 🔷 AI 编译器与软件栈

> 这一节的术语与 assembler 项目直接相关——assembler 本身就是一个 AI 编译器。

**IR（中间表示，Intermediate Representation）**
 编译器在源代码和目标机器码之间使用的中间形式。不依赖具体硬件，又足够形式化便于做分析和变换。你们项目的 FSIR → NNLayer → ZeusLayer → ABSNode → MachineInstr 就是 5 级 IR。

**Lowering（降级）**
 将高层 IR 变换成更低层、更接近硬件的 IR 的过程。每次 lowering 损失一些抽象性，换取更多硬件细节。你们项目从 FSIR 到 MachineInstr 的 5 步就是 5 次 lowering。

**Pass（编译步骤/编译趟）**
 编译器对 IR 执行的一次遍历与变换。可以是分析性的（如形状推断）或变换性的（如算子融合）。你们项目的 `FSIROptimizePass()`、`FSIR2NNGraphPass()` 等都是 Pass。

**DAG（有向无环图，Directed Acyclic Graph）**
 计算图的数学表示——节点是算子，边是数据流。你们项目的 ABSNode 层就是构建指令 DAG，表达哪些指令可以并行、哪些必须串行。

**ONNX（开放神经网络交换格式）**
 深度学习模型的标准表示格式，定义了跨框架的算子集和图格式。你们 compilerfrontend 的输入就是 ONNX 模型。

**TVM（张量虚拟机，Tensor Virtual Machine）**
 开源 AI 编译器框架，核心是 Relay IR + TIR + Auto-Tuning。你们的编译器栈借鉴了类似的多级 IR 思想。

**MLIR（多级中间表示，Multi-Level IR）**
 Google 提出的编译器基础设施，核心是可扩展的 Dialect（方言）机制。已成为现代 AI 编译器的主流基础框架。

**LLVM（底层虚拟机）**
 通用编译器基础设施，提供优化的 LLVM IR 和多架构后端代码生成。AI 编译器通常将 lowering 到 LLVM IR 作为对接 CPU/GPU 的最后一跳。

**Operator Fusion（算子融合）**
 将多个相邻算子合并成一个执行，减少中间结果写回内存的次数。例如 Conv+BN+ReLU 融合为一个操作。你们项目中 `fusion_info.json` 就是描述哪些算子被融合了。

**Tiling（分块）**
 将大张量分成适合片上存储（WL1/AL1）的小块，反复处理。你们项目中 `PE_CFG_DE` 指令就是在配置 tile 的切分细节。与你学 Triton 时的 `BLOCK_SIZE` 是同一个概念。

**Scheduling（调度）**
 决定计算在时间和空间上的执行策略：哪些先做哪些后做、哪些在 core0 做哪些在 core1 做。你们的 stschedule 就是做这件事。

**Auto-Tuning（自动调优）**
 通过搜索自动寻找最优的调度参数（tile size、并行策略等）。TVM 的 AutoTVM/Ansor 是典型实现。

**Kernel（核函数/计算核）**
 在加速器上执行的最小计算单元函数。一个算子（如矩阵乘法）对应一个或多个优化好的 kernel。

**Runtime（运行时）**
 负责在目标硬件上执行编译后模型的软件层——内存分配、kernel 调度、设备同步等。

**Driver（驱动程序）**
 操作系统内核中与硬件直接交互的模块。是用户态 Runtime 与物理硬件之间的桥。

**SDK（软件开发套件，Software Development Kit）**
 芯片厂商面向开发者的工具包：编译器 + Runtime + 示例代码 + 性能分析工具 + 算子库。你们项目输出的 `net_config_*.json` 就是给 SDK 用的接口文件。

**HAL（硬件抽象层，Hardware Abstraction Layer）**
 屏蔽底层硬件差异、向上提供统一接口的软件层。使上层软件无需关心具体硬件型号。

**BSP（板级支持包，Board Support Package）**
 针对特定硬件板子的底层软件包：bootloader + 驱动 + 硬件初始化代码。是嵌入式系统启动的基础。

**Firmware（固件）**
 固化在芯片 Flash/ROM 中的底层软件。MCU 的程序、加速器的微码（microcode）都算固件。

### 🔷 大模型 / Transformer 推理专用术语

> 这些术语在你的新任务（Qwen/Llama 大模型落地）中会反复出现。

**Self-Attention（自注意力）**
 Transformer 的核心计算：让序列中每个 token 与所有其他 token 计算相关性（Q×K^T），再对 V 加权求和。计算量 O(n²)，是大模型推理的主要瓶颈。

**MHA / MQA / GQA（多头/多查询/分组查询注意力）**
 MHA = 每个注意力头都有独立的 K、V，KV Cache 最大。MQA = 所有头共享一组 K、V，KV Cache 最小但精度略降。GQA = 折中，h 个头分成 g 组共享 K、V。Llama 3、Qwen2 等主流模型用 GQA。

**KV Cache（键值缓存）**
 自回归生成时，缓存已计算过的 K 和 V 矩阵，避免每生成一个 token 都重算整个历史。是 LLM 推理的核心优化。代价：显存随序列长度线性增长。

**Prefill（预填充阶段）**
 处理输入 prompt 的阶段：一次性处理所有 input token，计算并缓存 KV。特点：大矩阵乘法（GEMM），Compute-bound，硬件利用率高。

**Decode（解码阶段）**
 逐 token 生成输出的阶段：每次只生成 1 个新 token。特点：矩阵×向量（GEMV），Memory-bound（每个 token 要读全部权重+KV），硬件利用率低。这是 LLM 推理的主要瓶颈。

**TTFT（首 Token 延迟，Time To First Token）**
 从发请求到收到第一个输出 token 的时间。主要取决于 Prefill 耗时。用户感知最强的延迟指标。

**TPOT（每 Token 生成延迟，Time Per Output Token）**
 后续每个 token 的平均生成时间。决定输出文字"流速"的体验。

**Token / Tokenizer**
 Token = 语言模型处理的最小单位（可以是一个词、子词或字符）。Tokenizer = 将原始文本切分成 token 序列的模块。

**Context Length（上下文长度）**
 模型单次推理能处理的最大 token 数（prompt + 已生成 token 之和）。主流 LLM 已扩展到 128K~1M token。也是 KV Cache 显存的决定因素。

**RoPE（旋转位置编码，Rotary Position Embedding）**
 通过对 Q/K 向量做旋转变换来编码位置信息。Llama、Qwen 等主流模型均使用。外推性好（能处理比训练时更长的序列）。

**Flash Attention**
 将 Q/K/V 分块在片上 SRAM 内完成完整注意力计算，避免将巨大的注意力矩阵写回显存。大幅节省带宽和显存。已成为推理和训练的标配优化。

**PagedAttention（分页注意力）**
 vLLM 的核心技术：借鉴 OS 虚拟内存分页思想，将 KV Cache 分成固定大小的"页"动态分配，消除显存碎片化。

**Continuous Batching（持续批处理）**
 推理服务的调度策略：在序列级别动态增删请求，不等一个 batch 全完成再处理下一批。大幅提升吞吐量。

**Speculative Decoding（投机解码）**
 用小模型快速并行生成 k 个候选 token，再让大模型一次性并行验证。在不改变输出分布的前提下加速生成。

**Tensor Parallelism / Pipeline Parallelism / Data Parallelism**
 三种多卡并行策略：TP = 将单层权重切到多卡上并行算（层内并行，需高带宽互连）；PP = 将不同层分到不同卡上流水线执行（层间并行）；DP = 模型复制到每张卡，各处理不同数据。

**MoE（混合专家模型，Mixture of Experts）**
 用多个独立的"专家" FFN 替代单一 FFN，每个 token 只激活少数专家（如 Top-2）。总参数量大但每次只用一小部分。代表：Mixtral、DeepSeek-V2/V3。

**LoRA / QLoRA（低秩适配/量化低秩适配）**
 参数高效微调技术：冻结预训练权重，只训练注入的小矩阵（秩 r 的低秩分解）。QLoRA 更进一步将基础模型量化为 4-bit。让消费级 GPU 也能微调大模型。

**GPTQ / AWQ**
 两种主流的训练后权重量化算法。GPTQ 基于二阶信息做误差补偿量化；AWQ 通过分析激活值分布保护关键通道。都能在 3~4-bit 下保持很好的精度。

**W4A8 / W8A8（量化配置简写）**
 W = 权重位宽，A = 激活值位宽。如 W4A8 = 权重 4-bit、激活 8-bit。

**FP8（8 位浮点）**
 新的低精度浮点格式（E4M3/E5M2），NVIDIA H100 原生支持。兼具 INT8 的速度和更好的数值范围。

**MFU（模型浮点利用率，Model FLOPs Utilization）**
 实际吞吐对应的计算量与硬件理论峰值的比值。衡量"你用了多少硬件算力"。优秀的推理引擎 MFU 可达 50%+。

**Gradient Checkpointing（梯度检查点）**
 训练时不保存所有前向激活值，反向时按需重算。用额外计算换显存节省。

**Mixed Precision Training（混合精度训练）**
 前向/反向用 FP16/BF16 加速，梯度和主权重保留 FP32 保精度。NVIDIA AMP 是标准实现。与你们芯片 PE 的 INT8 乘法+FP32 累加是同一个混合精度思想。
